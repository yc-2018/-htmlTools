# 首页地球阳光明暗设计说明

## 目标

为首页点云地球增加轻微、连续的昼夜明暗变化。太阳方向固定在世界空间的画面右上方；地球自转时，不同区域自然经过亮面、晨昏过渡区和暗面。效果只增强球体感，不改变首页内容的视觉主次。

## 已确认的视觉方向

- 采用“平衡”强度：亮面最高为原色亮度的 `1.10` 倍，暗面最低为 `0.76` 倍。
- 明暗之间使用柔和曲线过渡，不显示生硬的半球切线。
- 太阳位于朝向画面右上方且略偏向观察者的固定方向。实现使用世界空间向量 `(0.55, 0.65, 1.0)`，归一化后传入着色器。
- 海洋、普通陆地和中国高亮点保留各自原色、透明度、大小和层级，只在原色上乘以光照亮度。
- 大气层保持现有均匀亮度，避免外缘随暗面消失。
- 月球沿用现有独立明暗表现；卫星、空间站、尾迹和轨道不受本次光照影响。

## 光照模型

地球的三个点云材质共享同一套 GPU 光照逻辑。顶点着色器将每个点的球面方向转换到世界空间，再与固定太阳方向求点积。这样 `globeGroup` 自转时，点的世界法线会持续变化，而太阳方向保持不动。

点积经过 `smoothstep(-0.25, 0.35, lightDot)` 转为昼夜混合值，再用该值在 `0.76` 与 `1.10` 之间插值。负侧较宽的过渡区使晨昏线柔和，也保证背光面仍清楚可见。

光照只改变 RGB。现有点透明度继续由各图层材质控制，深度球遮挡、透明混合和渲染顺序保持不变。自定义材质需要使用与当前渲染器一致的输出编码，避免替换 `PointsMaterial` 后原色发生非预期偏移。

## 代码结构

修改范围限制在 `js/homepage-earth.js` 和新增的聚焦测试文件内：

- 在现有 `config` 中增加太阳方向、亮面亮度、暗面亮度和晨昏线范围配置。
- 增加一个创建地球点云光照材质的函数，统一处理固定颜色图层和逐点颜色图层。
- `createPoints()` 继续负责几何体与颜色属性，只将地球点云的材质切换为新的光照材质。
- Three.js 自动提供的 `modelMatrix` 负责把球面方向变换到世界空间，不在 JavaScript 动画循环中逐点更新颜色或光照。
- 现有 `render()`、地球旋转、指针视差、响应式重建和 WebGL 上下文生命周期不改变。

现有工作区中被删除的测试文件属于用户改动，本功能不恢复或覆盖它们。新增测试使用 `tests/homepage-earth-lighting.test.js`，避免与这些删除项冲突。

## 性能与兼容性

- 桌面端仍使用 24,000 个候选点，移动端仍使用 14,000 个候选点。
- 每个顶点只增加一次向量变换、点积和插值，由 GPU 并行执行。
- 动画循环不新增数组分配、颜色缓冲更新或 CPU 逐点遍历。
- 继续兼容仓库内置 Three.js r128 和当前 Chrome、Edge。
- WebGL 初始化或运行环境不可用时，继续走现有 `earth-background-fallback` 回退，不增加新的回退分支。

## 测试与验收

自动化测试覆盖：

- 光照配置值、固定太阳方向和晨昏线范围符合本设计。
- 暗面亮度小于 `1`，亮面亮度大于 `1`，且任何点的亮度都限制在 `0.76` 至 `1.10` 之间。
- 地球点云使用世界空间法线与固定太阳方向计算光照。
- 海洋逐点颜色与陆地、中国固定颜色均进入同一套光照路径。
- 动画循环不包含逐点颜色重算，现有点数、透明度和旋转速度保持不变。

浏览器验收覆盖：

- 首次面对中国区域时能辨认右上方略亮、左下方略暗，但中国红色仍清楚。
- 地球旋转后，明暗区域固定在画面方向，地表区域会穿过晨昏线，光照不会粘在经纬度上一起旋转。
- 晨昏线过渡柔和，暗面仍可辨认海洋、陆地和中国点。
- 月球、卫星、空间站、轨道、大气层和首页文字观感不发生非预期变化。
- 桌面宽度和移动端宽度下动画流畅，控制台无错误或着色器警告。

## 非目标

- 不添加可见的太阳模型、太阳光晕或镜头光斑。
- 不按真实日期、时间或经纬度计算太阳位置。
- 不增加城市夜光、云层、阴影投射或新的外部纹理。
- 不调整地球尺寸、转速、初始朝向、点数或鼠标视差。
