# 首页地球真实陆地点阵设计

## 目标

修正首页地球点阵与现实世界地图轮廓明显不符的问题。地球只显示真实陆地点，不显示海洋点；中国区域使用浅红色，其他陆地区域统一使用浅灰色。现有地球缩放、旋转、指针视差、卫星和轨迹动画保持不变。

## 数据与结构

- 以 Natural Earth 低分辨率世界地图为基础，提取并适度简化陆地多边形坐标。
- 将简化后的坐标数据内嵌在 `js/homepage-earth.js`，不增加运行时网络请求或第三方依赖，页面仍可直接离线打开。
- 删除现有由十个椭圆组成的 `landRegions` 近似模型，以及海洋点阵生成逻辑。
- 中国大陆、海南和台湾归入中国高亮区域；其余陆地均归入普通陆地区域。

## 点阵生成

继续使用 Fibonacci sphere 在球面上均匀采样，但提高桌面端和移动端的候选点数量。每个候选点转换为经纬度后，通过真实陆地多边形进行筛选：

1. 位于中国区域时加入浅红色点阵。
2. 位于其他陆地区域时加入浅灰色点阵。
3. 位于海洋时不创建点。

中国点与其他陆地点采用相同的尺寸、密度和接近的透明度，仅通过颜色区分，避免中国轮廓显得突兀或像额外覆盖层。

## 朝向与视觉结果

保留以中国附近为初始正面的展示意图，并校准球体旋转，使中国上方能看到俄罗斯陆地点，下方按真实位置出现东南亚和海洋留白。南极洲只按其真实轮廓显示，不再使用整条 `latitude < -60` 的实心带状近似。

浅红色应比现有高亮更柔和；其他陆地点继续使用当前风格的浅灰色。地球外观仍是轻量背景，不增加国界线、海岸线描边或文字标签。

## 性能与兼容性

- 地图数据在脚本解析时冻结并复用，点阵只在桌面/移动断点切换时重建。
- 多边形判断先使用包围盒排除无关区域，再执行点在多边形内判断，控制候选点增多后的计算量。
- 不引入构建步骤，继续兼容当前 Chrome 和 Edge，并保留 WebGL 不可用时的回退逻辑。

## 验证

- 为地图判断补充自动化测试，覆盖中国、俄罗斯、印度、东南亚、澳大利亚、南美洲、非洲、海洋和南极洲的代表坐标。
- 验证海洋代表坐标不再生成点，中国坐标归入浅红色点阵，其他陆地坐标归入浅灰色点阵。
- 运行现有 `homepage-earth-background` 测试，确保动画、响应式配置和导出 API 没有回归。
- 在桌面和移动宽度下用 Chrome 或 Edge 查看首页，重点检查中国上下方的相对地理位置、陆地轮廓和点阵密度。

## 范围限制

本次只修改首页地球点阵的数据、分类、密度与初始朝向；不修改首页布局、卫星动画、交互方式或其他工具。
