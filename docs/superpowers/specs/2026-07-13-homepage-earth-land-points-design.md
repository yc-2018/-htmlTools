# 首页地球真实陆地点阵设计

## 目标

修正首页地球点阵与现实世界地图轮廓明显不符的问题。地球使用真实陆地轮廓，并以较低的白蓝色点阵表现海洋；中国区域使用更清晰的浅红色，其他陆地区域统一使用浅灰色。陆地点阵相对海洋轻微向球体外侧凸起，形成克制的三维层次。现有地球缩放、旋转、指针视差、卫星和轨迹动画保持不变。

## 数据与结构

- 以 Natural Earth 低分辨率世界地图为基础，提取并适度简化陆地多边形坐标。
- 将简化后的坐标数据内嵌在 `js/homepage-earth.js`，不增加运行时网络请求或第三方依赖，页面仍可直接离线打开。
- 删除现有由十个椭圆组成的 `landRegions` 近似模型，继续使用真实陆地多边形区分陆地与海洋。
- 中国大陆、海南和台湾归入中国高亮区域；其余陆地均归入普通陆地区域。

## 点阵生成

继续使用 Fibonacci sphere 在球面上均匀采样，但提高桌面端和移动端的候选点数量。每个候选点转换为经纬度后，通过真实陆地多边形进行筛选：

1. 位于中国区域时加入较清晰的浅红色点阵。
2. 位于其他陆地区域时加入浅灰色点阵。
3. 位于海洋时加入更小、更淡的白蓝色点阵。

中国点与其他陆地点采用相同的采样密度和球面高度，但使用更大的点和更高的颜色对比。中国颜色调整为 `#d94f5c`，透明度约为 `0.9`，点大小约为 `2.1` 像素，确保完整自转过程中转到球缘时仍能辨认。

## 陆海三维分层

- 陆地点阵使用约 `1.008` 的球面半径，点大小约为 `1.75` 像素。
- 海洋点阵使用约 `0.995` 的球面半径，点大小约为 `1.4` 像素，透明度约为 `0.5`，避免小点经过抗锯齿后消失在浅色背景中。
- 海洋使用全部非陆地候选点，不再进行隔点降采样；颜色在 `#cceeff` 与 `#4fa6c8` 之间确定性混合，不随刷新随机变化。
- 深度遮挡球缩小到海洋点阵内侧，避免海洋点被遮挡或产生深度闪烁。
- 陆海高度差只用于增强层次，不添加实体国界线、海岸线描边或明显的浮雕边缘。

## 朝向与视觉结果

保留以中国附近为初始正面的展示意图，并校准球体旋转，使中国上方能看到俄罗斯陆地点，下方按真实位置出现东南亚和较低的白蓝色海洋层。南极洲只按其真实轮廓显示，不再使用整条 `latitude < -60` 的实心带状近似。

中国红色应在正面和球缘位置都清晰可辨；其他陆地点继续使用当前风格的浅灰色，海洋使用明显但不压过陆地的白蓝色。地球保持完整自转，外观仍是轻量背景，不增加国界线、海岸线描边或文字标签。

## 性能与兼容性

- 地图数据在脚本解析时冻结并复用，点阵只在桌面/移动断点切换时重建。
- 多边形判断先使用包围盒排除无关区域，再执行点在多边形内判断，控制候选点增多后的计算量。
- 不引入构建步骤，继续兼容当前 Chrome 和 Edge，并保留 WebGL 不可用时的回退逻辑。

## 验证

- 为地图判断补充自动化测试，覆盖中国、俄罗斯、印度、东南亚、澳大利亚、南美洲、非洲、海洋和南极洲的代表坐标。
- 验证海洋代表坐标归入全密度白蓝色低层点阵，中国坐标归入高对比红色高层点阵，其他陆地坐标归入浅灰色高层点阵。
- 验证陆地半径大于海洋半径，陆地点大小大于海洋点大小，深度遮挡球位于海洋层内侧。
- 验证中国点大小大于普通陆地点，海洋生成逻辑不再使用 `index % 2` 降采样。
- 运行现有 `homepage-earth-background` 测试，确保动画、响应式配置和导出 API 没有回归。
- 在桌面和移动宽度下用 Chrome 或 Edge 查看首页，重点检查中国上下方的相对地理位置、陆地轮廓和点阵密度。

## 范围限制

本次只修改首页地球点阵的分类、颜色、密度、点大小和陆海球面高度；不修改首页布局、卫星动画、交互方式或其他工具。
