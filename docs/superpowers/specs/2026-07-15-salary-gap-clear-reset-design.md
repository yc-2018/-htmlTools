# 工资差距计算器清空与重置按钮设计说明

## 目标

在“工资不止差一半计算器”的“每月工资”标题区增加“清空”和“重置”两个按钮，帮助用户快速重新输入或恢复默认示例。按钮必须保持紧凑，不挤占额外操作行，并与现有实时计算、花销同步锁和 URL 分享规则一致。

## 位置与视觉

- 采用用户确认的标题栏方案。
- “每月工资”、 “清空”和“重置”始终位于同一行；按钮不单独占一行。
- 桌面端标题与两个按钮组成左侧操作组，原有“甲乙工资各自独立”说明保持在标题栏右侧。
- 手机端标题与两个按钮仍保持同排；说明文字可放在该行下方，避免压缩按钮或产生横向滚动。
- “清空”为浅色描边按钮，“重置”为深色实心按钮，沿用页面暖色与深绿色配色，不使用截图中的红色标注框作为正式样式。
- 两个按钮使用紧凑字号和间距，最小触控高度为 40 像素，并提供明确的悬停与键盘焦点状态。

## 清空行为

点击“清空”后：

1. 清空甲工资、乙工资、甲花销和乙花销四个输入框。
2. 保留当前花销锁定状态；锁定时两侧保持同步的空值，解锁时两侧均独立为空。
3. 复用现有无效输入渲染：月、年和 10 年结果显示 `—`，结论提示“请输入有效的非负金额”。
4. 不更新 URL；地址栏保留清空前最后一次完整有效参数，符合现有“无效输入不覆盖分享链接”的规则。

## 重置行为

点击“重置”后：

1. 甲工资恢复为 `5000`，乙工资恢复为 `10000`。
2. 甲乙花销恢复为 `4000`。
3. 花销同步锁恢复为锁定状态。
4. 结果恢复为默认的 `1000 / 6000` 月剩余和 `6 倍`结论。
5. 通过现有 `history.replaceState` 同步完整默认参数，不新增浏览器历史记录；未知参数和 `#hash` 继续保留。

重置值来自页面初始化前读取的 HTML 默认值，而不是当前 URL 参数，因此从任何分享链接点击“重置”都会回到产品默认示例。

## 代码边界

- `工资不止差一半计算器/index.html`：在工资标题中新增一行内操作组及两个原生按钮，补充响应式样式。
- `工资不止差一半计算器/app.js`：注册清空与重置点击事件，复用现有 `renderAndSync()`、`defaults` 和 `expenseState`。
- `工资不止差一半计算器/calculator.js`：无需修改。
- `tests/salary-gap-calculator.test.js`：扩展假 DOM 元素和控制器交互测试，增加页面静态契约断言。

## 可访问性

- 两个控件均使用带可见文字的 `<button type="button">`。
- DOM 顺序为“每月工资”“清空”“重置”，键盘 Tab 顺序与视觉顺序一致。
- 使用 `:focus-visible` 提供高对比焦点环。
- 不仅依靠颜色区分操作，按钮文字始终可见。

## 测试与验收

自动化测试覆盖：

- 清空四个输入并保留锁状态。
- 清空后结果进入无效状态，URL 的 `replaceState` 调用次数不增加。
- 解锁状态下清空后仍保持解锁。
- 重置恢复四个默认值和锁定状态。
- 重置后 URL 写入默认参数且不调用 `pushState`。
- HTML 中按钮 ID、`type="button"`、一行操作组和手机端样式存在。

浏览器验收覆盖：

- 桌面端和手机端“每月工资 / 清空 / 重置”保持同一行。
- 320 像素窄屏无横向滚动，按钮不重叠，触控高度达标。
- 清空、重置、锁状态、计算结果和地址栏行为与规格一致。
- 键盘可操作且焦点清晰，控制台无错误或警告。

## 非目标

- 不增加二次确认弹窗。
- 不增加撤销按钮或独立历史记录。
- 不改变计算公式、默认说明文字或结果布局。
