# 工资不止差一半计算器设计说明

## 目标

新增一个无需构建、可直接在现代浏览器中打开的原生 HTML 工具，用固定支出前后的倍数差说明“工资差距不等于可支配金额差距”。页面必须同时适配桌面端和手机端，并随输入实时更新。

## 页面入口与文件

- 新工具目录：`工资不止差一半计算器/`
- 页面入口：`工资不止差一半计算器/index.html`
- 可复用计算逻辑：`工资不止差一半计算器/calculator.js`
- 自动化测试：`tests/salary-gap-calculator.test.js`
- 在根目录 `index.html` 的“生活与计算”列表中添加工具入口。

页面使用原生 HTML、CSS 和 JavaScript，不引入框架、字体、图标库或网络资源。CSS 放在页面内；纯计算函数放在 `calculator.js` 中，供页面与 Node 测试共同使用。

## 内容与视觉结构

页面采用已确认的“A：左右对照卡片”方案。

1. 顶部标题为“工资不止差一半”。
2. 标题下展示说明文字：甲月薪 5000 元、乙月薪 10000 元，两人房租、水电、吃饭等固定支出均为 4000 元；甲每月剩余 1000 元，乙每月剩余 6000 元，因此工资表面是 2 倍，可支配金额实际是 6 倍。
3. “每月工资”区域左右放置甲、乙两个独立数字输入框，默认值分别为 `5000`、`10000`。
4. “每月固定消耗”区域左右放置甲、乙两个数字输入框，默认值均为 `4000`；锁按钮只位于这两个消耗输入框之间。
5. 结果区按“每月、每年、10 年”三行展示甲乙剩余金额。桌面端使用紧凑对照表，窄屏下保持清晰的甲乙两列卡片关系，不产生横向滚动。
6. 底部使用深色强调卡展示倍数结论，默认文案为：

   > 你以为乙的工资是甲的 2 倍？
   >
   > 可事实却是 **6 倍**

   第二行和最终倍数使用更大的字号及高对比强调色。

视觉使用暖白背景、低饱和绿色表示甲、暖橙色表示乙、深绿色结论卡。控件必须保持足够对比度和触控面积，手机端输入框与锁按钮不小于约 44 像素高。

## 交互规则

### 工资输入

- 甲乙工资互不联动。
- 输入变化时立即重新计算，不需要“计算”按钮。

### 固定消耗同步锁

- 初始状态为锁定。
- 锁定时，修改任意一侧固定消耗，另一侧立即同步为相同值。
- 点击锁按钮后解锁，两侧固定消耗可独立修改。
- 再次点击后恢复锁定，并立即以甲的固定消耗覆盖乙的固定消耗。
- 按钮使用内联 SVG 锁图标，不依赖 Emoji 或外部图标；通过 `aria-pressed`、可见状态文字或无障碍名称表达当前状态和点击后的动作。

## 计算规则

对甲、乙分别计算：

```text
月剩余 = 月工资 - 每月固定消耗
年剩余 = 月剩余 × 12
10 年剩余 = 年剩余 × 10
```

计算不包含利息、投资收益、通胀、涨薪或支出变化。

金额使用中文数字分组格式，最多保留两位小数；整数不强制显示小数。倍数最多保留两位小数，并移除无意义的末尾零。

### 倍数结论

- 工资不相等且较小工资大于 0 时，第一行按工资较高者在前，显示工资倍数。
- 较小工资为 0 时，不计算工资倍数，第一行显示“工资倍数无法计算”。
- 两人的月剩余均大于 0 时，第二行按月剩余较大者在前计算可支配金额倍数；默认值下简化显示为“可事实却是 6 倍”。
- 如果工资高低与月剩余高低不是同一个人，第二行明确写出可支配金额较高者，避免暗示错误。
- 工资相同时，第一行改为“两人的工资相同”。
- 任意一方月剩余小于或等于 0 时，不对可支配金额计算倍数；结论改为“已入不敷出，无法用倍数衡量”。
- 两人的正月剩余相同时，显示“两人的可支配金额相同”。

## 输入与异常状态

- 工资和固定消耗只接受大于或等于 0 的有限数字，允许最多两位小数。
- 输入框暂时为空或包含无效值时，不把它静默转换为 0；对应结果显示 `—`，结论区提示“请输入有效的非负金额”。
- 月剩余为 0 或负数时，金额仍按真实结果展示，并使用警示色和“已入不敷出”标记。
- 页面不会弹出阻塞式提示框，所有状态都在结果区就地反馈。

## 响应式与可访问性

- 桌面端容器限制最大宽度，甲乙输入与结果保持左右对照。
- 手机端在不破坏甲乙对应关系的前提下缩小间距和字号；说明文字、输入区、结果区与结论卡按自然顺序纵向排列。
- 输入框具有明确的 `<label>`，金额单位以可见文本呈现。
- 锁按钮可用键盘操作，焦点样式清晰；颜色不是区分锁定状态或异常状态的唯一手段。
- 使用语义化标题、分区和结果状态文本；实时结果区域使用不会频繁打断读屏的适当 `aria-live` 设置。
- 尊重 `prefers-reduced-motion`，仅使用轻量过渡，不依赖动画传达信息。

## 测试与验收

### 自动化测试

使用仓库现有的 Node 测试方式覆盖纯计算函数：

- 默认值计算为月剩余 `1000/6000`、年剩余 `12000/72000`、10 年剩余 `120000/720000`。
- 默认工资倍数为 `2`，默认可支配金额倍数为 `6`。
- 小数金额与格式化规则正确。
- 两人工资或月剩余相同的结论分支正确。
- 月剩余为 0、负数及输入无效时不会产生 `Infinity`、`NaN` 或误导性倍数。
- 支出解锁后出现可支配金额高低反转时，结论中的人物顺序正确。

### 浏览器验收

- 验证初始默认值、实时更新以及锁定状态下双向同步。
- 验证解锁后独立输入，重新锁定时以甲覆盖乙。
- 验证键盘操作、焦点状态、无效输入和入不敷出提示。
- 在桌面视口和典型手机视口中检查无横向滚动、文字不重叠、触控区域足够大。
- 检查浏览器控制台无错误或警告。

## 非目标

- 不保存历史记录或写入本地存储。
- 不支持税费、奖金、利息、通胀、涨薪等高级模型。
- 不提供图表、导出、分享或后端服务。
