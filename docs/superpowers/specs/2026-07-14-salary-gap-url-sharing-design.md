# 工资差距计算器 URL 分享设计说明

## 目标

让“工资不止差一半计算器”的完整输入状态体现在地址栏中。用户复制当前 URL 后，其他人打开链接即可恢复相同工资、花销、同步锁状态和计算结果。输入过程中不得为每次变化新增浏览器历史记录。

## URL 参数契约

页面使用以下可读查询参数：

| 参数 | 含义 | 默认值 |
| --- | --- | --- |
| `salaryA` | 甲的每月工资 | `5000` |
| `salaryB` | 乙的每月工资 | `10000` |
| `expenseA` | 甲的每月固定消耗 | `4000` |
| `expenseB` | 乙的每月固定消耗 | `4000` |
| `locked` | 固定消耗是否同步，`1` 为锁定、`0` 为解锁 | `1` |

默认分享链接形如：

```text
?salaryA=5000&salaryB=10000&expenseA=4000&expenseB=4000&locked=1
```

金额参数沿用计算器现有规则：必须是大于或等于 0 的有限十进制数字，最多两位小数。URL 中不存在或不合法的单个金额参数不影响其他合法参数，该项使用 HTML 中的默认值。`locked` 只有 `0` 和 `1` 有效，其他值使用默认锁定状态。

页面保留不属于本工具的未知查询参数和现有 `#hash`，只新增或覆盖上述五个自有参数。

## 初始化规则

1. 页面取得 HTML 输入框的默认值。
2. 解析当前 URL，将每个合法参数分别覆盖到对应默认值上。
3. `locked=0` 时保留甲乙两侧的独立花销。
4. `locked=1` 时立即以 `expenseA` 为准同步 `expenseB`，即使传入链接中的两侧花销不同。
5. 初始化计算器、渲染结果和锁按钮状态。
6. 使用标准化后的有效状态调用一次 `history.replaceState`，使无参数链接、缺失参数链接和部分非法链接都变成可直接分享的完整链接。

## 实时更新规则

- 工资或花销发生有效变化时，计算结果更新后立即同步 URL。
- 锁定状态切换后立即同步 URL；重新锁定仍沿用现有“以甲覆盖乙”的规则，再把同步后的两侧花销写入参数。
- 使用 `history.replaceState` 更新当前地址，不调用 `pushState`，因此每次键入不会产生新的浏览器历史记录。
- 输入暂时为空、为负数、超过两位小数或其他无效状态时，页面仍显示现有校验错误，但不覆盖 URL；地址栏保留上一次完整有效状态。
- 当输入重新有效后，URL 立即恢复同步。

## 代码边界

- `工资不止差一半计算器/calculator.js`：扩展花销联动状态构造函数，使其可以接收初始锁定状态；锁定初始化时以甲为准。
- `工资不止差一半计算器/app.js`：增加 URL 状态解析、URL 构造和地址栏替换逻辑；浏览器自动初始化时注入 `location` 与 `history`，Node 测试继续使用可控的假对象。
- `工资不止差一半计算器/index.html`：不新增按钮或视觉组件。
- `tests/salary-gap-calculator.test.js`：扩展现有纯函数与 DOM 控制器测试。

URL 解析与构造应保持独立、可测试；DOM 控制器只负责按顺序读取状态、渲染和触发地址栏更新。

## 异常与兼容性

- 如果运行环境没有 `history.replaceState`，计算器继续正常工作，只跳过地址栏同步。
- URL 解码和参数编码使用浏览器内置 `URL` 与 `URLSearchParams`，不手写编码规则。
- 旧的无参数链接保持兼容，打开后使用原有默认值并补齐参数。
- 不监听或记录每次输入历史；浏览器前进、后退仍按普通页面导航工作。

## 测试与验收

自动化测试覆盖：

- 完整参数恢复四个输入值和解锁状态。
- 缺失或非法参数逐项回退，不污染其他合法参数。
- 锁定链接以甲的花销覆盖乙，解锁链接保留不同花销。
- 默认页面初始化后生成完整参数。
- 有效输入和锁切换调用 `replaceState`，从不调用 `pushState`。
- 无效输入不改变上一次有效 URL，重新输入有效值后恢复更新。
- 未知参数与 hash 保留，金额按 URL 标准正确编码。

浏览器验收覆盖：

- 修改工资、锁定花销、解锁后修改不同花销时，地址栏参数实时变化。
- 复制当前链接到新标签页可恢复完全相同的界面和计算结果。
- 连续输入后按一次后退，不会逐字符回退。
- 控制台无新增错误或警告。

## 非目标

- 不增加“复制链接”或“分享”按钮。
- 不使用本地存储、Cookie 或服务端短链接。
- 不为每次输入创建可后退的历史快照。
