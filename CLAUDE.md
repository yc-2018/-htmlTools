## 项目概述

这是一个使用纯 HTML、CSS 和 JavaScript 构建的 HTML 工具和迷你游戏集合。该仓库包含三个主要类别：

1. **实用工具** - 各种基于 Web 的开发工具、编码工具、计算器等
2. **图一乐** - 迷你游戏和交互式演示集合
3. **没啥用的** - 实验性或纪念性项目

## 开发命令

这是一个静态 HTML/JS 项目，大多数工具不需要构建系统。但某些项目有特定的构建要求：

### 3D 蜘蛛网项目 (图一乐/3D蜘蛛网/)
```bash
# 进入项目目录
cd "图一乐/3D蜘蛛网"

# 安装依赖
npm install

# 构建项目
npm run build
```

### 一般开发
- 所有工具都是独立的 HTML 文件，可以直接在浏览器中打开
- 大多数项目不需要编译或构建步骤
- 开发时直接编辑 HTML、CSS 或 JS 文件即可

## 技术栈

- **前端**: HTML5, CSS3, 原生 JavaScript
- **框架**: 部分项目使用 Bootstrap、Vue.js (css网格布局)、React (react-amap)
- **物理引擎**: Verlet.js 用于 3D 蜘蛛网物理效果
- **构建工具**: Browserify 和 UglifyJS (仅用于 3D 蜘蛛网)

## 关键文件和入口点

- `index.html` - 主导航页面
- 各个工具目录包含自己的 `index.html`
- 大多数游戏是 `图一乐/小游戏/` 中的单个 HTML 文件

## 添加新工具

1. 为新工具创建新目录
2. 添加包含工具功能的 `index.html` 文件
3. 更新主 `index.html` 以包含指向新工具的链接
4. 遵循现有的自包含 HTML/CSS/JS 文件模式

## 代码风格指南

- 大多数项目都包含在单个 HTML 文件中
- CSS 通常嵌入在 `<style>` 标签中或单独的 `.css` 文件中
- JavaScript 通常在单独的 `.js` 文件中或嵌入在 HTML 中
- 整个项目使用中文注释和变量名
- Bootstrap 通常用于响应式布局

## 测试

- 没有设置自动化测试框架
- 在浏览器中打开 HTML 文件进行手动测试是标准方法
- 所有工具应在现代浏览器（Chrome、Firefox、Safari、Edge）中正常工作

## 部署

这是一个可以部署到任何 Web 服务器的静态网站：
1. 将所有文件复制到 Web 服务器
2. 确保保持目录结构
3. 通过 `index.html` 作为入口点访问
