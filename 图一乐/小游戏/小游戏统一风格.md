# 小游戏统一风格

这份规范用于这一组轻量 HTML 小游戏，目标是让不同玩法在菜单、暂停、记录、HUD 和触控体验上保持一致。

## 视觉基调

- 页面使用全屏沉浸式布局，`html`、`body` 固定为 100% 宽高并隐藏滚动。
- 背景使用深色多层渐变，主色以深紫黑、暖黄、青绿和少量粉红组成。
- UI 面板使用半透明玻璃感：细边框、低透明背景、较重阴影和 `backdrop-filter: blur(...)`。
- 圆角统一克制，主要容器和按钮保持 `8px` 左右。
- 标题字重高，字号只在真正的菜单和结果页放大；HUD 和按钮保持紧凑。

## HUD

- HUD 固定在顶部安全区内，使用小型统计盒展示核心数值。
- 每个统计盒包含一个浅色小标签和一个高权重数值。
- 暂停按钮放在 HUD 右侧，尺寸约 `42px`，使用符号 `Ⅱ`，只负责进入暂停。
- 暂停后继续操作放在暂停菜单里完成，顶部暂停按钮不切换成继续按钮。

## 屏幕面板

所有菜单、暂停和结算层都使用同一结构：

```html
<div class="screen hidden" id="pauseScreen">
  <section class="panel">
    <div class="kicker">Paused</div>
    <h1>已暂停</h1>
    <button class="primary">继续</button>
    <button class="ghost">重新开始</button>
    <button class="ghost">返回菜单</button>
  </section>
</div>
```

推荐样式要点：

```css

.screen.hidden{opacity:0;visibility:hidden;pointer-events:none}
.panel{width:min(420px,90vw);padding:24px 20px;border:1px solid rgba(255,255,255,.16);border-radius:8px;background:rgba(31,27,38,.88);box-shadow:0 28px 86px rgba(0,0,0,.44);text-align:center;backdrop-filter:blur(18px)}
```

## 模式与记录

- 每个游戏提供 3 个模式，使用分段按钮切换。
- 菜单显示文案统一为“当前模式最高分”。
- 最高分按模式分别保存，不使用单个全局最高分覆盖所有模式。
- localStorage 推荐保存为 JSON：

```js
const BEST_KEY = 'game_name_best_by_mode_v1';

function readBestScores(){
  const scores = {easy:0, normal:0, hard:0};
  try{
    const saved = JSON.parse(localStorage.getItem(BEST_KEY) || '{}');
    for(const key of Object.keys(scores)) scores[key] = Number(saved[key] || 0);
  }catch(e){
    localStorage.removeItem(BEST_KEY);
  }
  return scores;
}
```

- 结算、暂停返回菜单、暂停重新开始前，都要先记录当前模式最高分。
- 如果旧版本有单一最高分，可以迁移到 `normal` 模式。

## 状态与交互

- 状态建议统一为 `menu`、`playing`、`paused`、`over`。
- `pauseGame()` 只在 `playing` 时生效，并显示暂停菜单。
- `resumeGame()` 只在 `paused` 时生效，并隐藏暂停菜单。
- `showMenu()` 从暂停菜单进入主菜单时应直接切换，不再展示结算层。
- 移动端必须禁用默认滚动和选择，画布使用 `touch-action:none`。
- 游戏循环在非 `playing` 状态下可以继续绘制背景，但不推进玩法逻辑。
